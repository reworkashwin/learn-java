# Module Introduction: Practice Project — Building a Quiz App

## Introduction

Time for a fresh start! In this section, we're building a **quiz application** from the ground up. This isn't just a toy project — it's a carefully designed practice exercise that brings together everything we've learned about side effects, `useEffect`, dependencies, cleanup functions, and core React concepts like components and state.

---

## What You'll Practice

This project is designed to reinforce several key skills:

### 1. Side Effects & `useEffect`
You'll continue working with effects in a new context, deepening your understanding of when and how to use them.

### 2. Effect Dependencies & Cleanup Functions
You'll encounter real scenarios where understanding dependencies and cleanup is critical for correct behavior.

### 3. Combining Effects with Other React Concepts
Effects don't exist in isolation. You'll see how they interact with component composition, state management, props, and more.

---

## The Project Approach

This project is built **step by step**. While you can try building it on your own as a challenge, the primary flow is:

1. The instructor describes what feature to build next
2. You can **pause and try it yourself** before seeing the solution
3. Then follow along to understand the reasoning behind each decision

This is a great balance of guided learning and independent practice.

---

## Getting Started

If you're using the local project:
```bash
npm install
npm run dev
```

The starting screen is nearly empty — just a blank canvas waiting for us to bring the quiz to life.

---

## What We'll Build

A quiz app with:
- Questions displayed one at a time
- Multiple answer choices per question
- Tracking of user answers
- Timer-based auto-progression
- Results display at the end

Each of these features will exercise different React concepts, especially `useEffect` and its related patterns.

---

## ✅ Key Takeaways

- This is a **practice project** designed to solidify your understanding of `useEffect` and side effects
- Building from scratch reinforces component design, state management, and effect patterns
- Pausing to try each step on your own is the best way to learn
- The project combines multiple React concepts, not just effects

## 💡 Pro Tip

When you're following along, resist the urge to just copy code. Before looking at the solution, spend 2-3 minutes thinking about *how you'd approach it*. Even if you get stuck, that thinking time is when real learning happens.
