# Module Introduction — Adding Authentication to React Apps

## Introduction

Authentication is one of those features that turns a demo project into a real application. In this section, you'll learn how to **authenticate users** in React apps — from the ground up.

---

## What You'll Learn

### Core Authentication Concepts
- **How authentication works** in React applications — the underlying mechanism, not just the code
- **Implementing user signup and login** — sending credentials, handling responses

### Token Management
- **Storing authentication tokens** so the app remembers who's logged in
- **Attaching tokens to outgoing requests** to access protected backend resources

### User Experience
- **Auth persistence** — keeping track of whether a user is logged in across page reloads
- **Automatic logout** — expiring sessions after a set time for security
- **Updating the UI** based on authentication status (show/hide buttons, protect routes)

### New Routing Concepts
- Along the way, you'll also pick up a couple of **new React Router features** that fit naturally into authentication workflows

---

## Why This Matters

Almost every real application needs authentication. Whether it's a social media platform, an e-commerce site, or an internal dashboard — users need to log in, their sessions need to persist, and certain pages need protection.

By the end of this section, you'll have a complete authentication flow integrated into a React Router-based application.
