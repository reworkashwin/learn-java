# Getting The Most Out Of This Course

## Introduction

Having lots of content is great, but only if you know how to use it effectively. Let's cover the strategies and resources that will help you truly master React, not just watch videos about it.

---

## Prerequisites

This is important — for this course, you need **basic web development and JavaScript knowledge**.

- There's an optional JavaScript refresher section coming up next
- That refresher is for people who haven't used JavaScript in a while
- But it is **not** a replacement for a full JavaScript course

If you've never written JavaScript at all, take a JavaScript basics course first. But if you have at least basic knowledge — that's enough. No expert-level experience is required.

---

## How to Watch the Videos

This is a **video on-demand** course. Take advantage of that:

- **Slow down** if the pace is too fast
- **Speed up** if you already know a concept
- **Pause and think** when something doesn't click right away
- **Rewatch** lectures or even entire sections — this is a normal part of learning
- Don't rush. There's no deadline.

---

## Practice Makes Perfect

This is where most learning actually happens. Watching videos alone won't make you a React developer.

Here's what to do:

1. **Complete the coding exercises** — they're scattered throughout the course
2. **Follow along with demo projects** — type the code yourself, don't just watch
3. **Pause and try things on your own** — when building a project together, pause the video and attempt the next step yourself
4. **Build your own projects** — create dummy projects inspired by the demos or by real websites you've seen

The more you build, the faster you learn.

---

## When You Get Stuck

Getting stuck is inevitable and completely normal. Here are your resources:

### Code Snapshots
All course sections have code snapshots available on GitHub. Each snapshot corresponds to a specific lecture, so you can compare your code with the instructor's code to find issues.

### Q&A Section
Ask questions when you're stuck. But also **answer other students' questions** — this forces you to think deeply about problems and apply your knowledge, which is one of the best ways to learn.

### Discord Community
You also get access to a Discord server with other students. Learning together is more fun and more effective than learning alone.

---

## ✅ Key Takeaways

- Basic JavaScript knowledge is a prerequisite — the refresher section is helpful but not a substitute
- Watch at your own pace — pause, rewind, speed up as needed
- **Practice is non-negotiable** — build things, do exercises, try on your own
- Use code snapshots, Q&A, and Discord when stuck
- Answering others' questions is one of the best ways to solidify your own understanding

---

## ⚠️ Common Mistake

Watching all the videos without coding along. This creates an **illusion of understanding** — you think you know React because the explanations make sense, but when you try to build something on your own, you're lost. Always code along.

---

## 💡 Pro Tip

When a demo project is being built in the course, pause the video after the task is explained and **try to implement it yourself** before watching the solution. Even if you fail, the attempt makes the solution stick much better.
