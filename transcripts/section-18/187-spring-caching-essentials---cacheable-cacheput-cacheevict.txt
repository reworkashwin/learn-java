The new concept that we're going to focus is Caching. As a back-end developer, you should also be comfortable on how to implement caching inside the back-end business logic. What is caching? Caching means storing frequently used data inside the memory so that the next time the application can reuse the data instead of recalculating or refetching it from the database. By adopting the caching, we can improve the performance of an application to a great extent. Off course, we can't use caching for each and every scenario. You need to be very cautious on implementing this caching for a given scenario. There are certain requirements that you need to be aware whenever you want to implement caching. I'm going to explain that in a few minutes. First, let me answer a question that you may have, which is how come caching is going to improve the performance of the application? For that, I'm going to take a scenario, let's imagine there is a back-end database and inside this database, we have various kinds of data. And usually this database will be deployed in a separate server location. And your back-end application is going to be deployed in some other server. These two servers, one is which is having your application code and the other one, which is having database installed,they both can be present at two different countries or two different continents or within the same city. So regardless of where the servers are deployed, in a typical scenario, when a client application is making a request to the back-end REST API, this back-end REST API is going to rely on the database to fetch the required data. So to fetch that data, the back-end REST API, it needs to make a call to the database server, and this call has to travel over the network. So that's why whenever you want to perform database operation or any complex operation, usually they're going to take a few milliseconds of time. So it can be 50 millisecond or a hundred millisecond. As the amount of traffic that your application is handling is going to increase, the dependency on the database server is also going to increase. And with that, there is a very good chance the performance of your application may get reduced, that's why we need to rely on the caching wherever applicable. In the scenario of caching, what is going to happen is, only during the first request that data will be fetched from the database followed by the data will be stored inside the memory. When I say memory, this memory is RAM memory. So there are famous products around cache like Redis. What all these products they will do is, they will store that data inside the RAM memory so that when the next request comes, the data will be fetched from the cache instead of the database. Reading the data from the memory is going to take only microseconds. Whereas with the database scenario, it is going to take milliseconds of time. So there will be thousands of times of improvement because when we are trying to fetch the data from the database, the request has to travel through the network and the database server has to read the data from the disk. At the end of the day, all these databases, they are going to store the data inside the disk space or inside the file system. So reading the data from the disk space is more costlier compared to reading the data from the RAM memory location. These are the three common caching annotations that are used by developers on their day-to-day work. The first one is @Cacheable. By using this annotation, we're telling to the framework, please read that data from the cache if it is present,otherwise you can fetch from the database. For the first time, the framework is going to save that data into the cache memory so that all future calls or future requests can be provided with the cached data. Next, we have @CachePut annotation. We're going to use this annotation to tell to the framework that whenever a given method or a given logic is being executed, please update my cache records. In the similar lines, we have @CacheEvict as well. By using this annotation, we're going to tell to the framework, remove entry from the cache or reset the cache whenever a method that is annotated with this annotation is being executed. @Cacheable annotation, it is going to support POST and GET operation. When I say POST, it is going to help us in saving the data into the cache and reading the data from the cache.
