Before I try to close this lecture, let me try to revise the differences between singleton and prototype one more time. We know inside spring framework, the default scope is singleton, whereas to use the prototype scope, you need to explicitly define the @Scope annotation along with this constant. When we are using singleton scope, the same object instance will be returned every time we refer a bean inside the code, whereas with the prototype scope, a new object instance will be returned every time we refer a bean inside the application code. When a bean is having singleton scope, we can control when the bean is getting created, whether it should be during the startup or during when the first time being referred. Whereas with the prototype scope, we don't have to decide with the lazy or eager because it is going to give a brand new instance every time someone asks. That's why these prototype scope beans, they're not going to be initialized during the startup. For this scope, there is no concept of eager or lazy. You need to consider the singleton scope for the immutable objects. So what is the meaning of immutable object? Once the object is created, if you don't want to change anything inside that object and everyone should have the same copy for such scenarios, you can go with the singleton whereas you should use the prototype scope for the mutable object. Mutable objects means even after creating the object, we should be able to update the data inside that object. When we are using prototype scope, this should not create any problem because whatever data that is changed inside the mutable object, that is going to be specific to a user or a customer. Since each user is going to get their own copy of this prototype bean, even if they change some data inside the bean, it is going to work and reflect in their own session itself. Most of the times you should be fine with the singleton scope. But if there are some scenarios where you want to create a bean that is carrying some user specific data, please feel free to use the prototype bean scope.
