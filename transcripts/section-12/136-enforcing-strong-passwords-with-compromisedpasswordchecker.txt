Inside enterprise applications, when an end user is trying to register a new account, we usually try to enforce some conditions on the password that the end user is choosing. Because we don't want our end users to choose weak passwords. Due to weak passwords, there is a very good chance that a given account may get hacked easily by the hackers because weak passwords can be easily guessed by the hackers. It is our responsibility as a backend developer as well, to stop the end users from choosing a weak password. Off course, on the UI side, inside the registration page, there will be enough validations added by the UI developers like the password should be of at least 8 characters and it should have a combination of small letter, capital letter, one special character and one digit. Even with these kind of enforcements, but still end users, they end up choosing some weak passwords. For example, they may choose a password like password@1. This is a strong password according to the UI validations. But do you think it is a strong password? Off course not. This can be easily guessed by the hackers. That's why I strongly recommend you adding some backend validation as well to enforce the end user from choosing a meaningful strong password.
