Spring boot allows us to externalize the configurations or properties so that the same code works in different environments by leveraging different property values. So what is externalizing? Externalizing is a concept by using which you're going to feed the property value from outside of your source code. That's why this name externalizing has been given to this concept. By default, when we are not following this externalizing properties concept, the spring boot framework is going to look for the properties and their values inside the application.properties file or application.yml file. In case if you are using @PropertySource annotation, it is going to look into the custom property file that you have configured. But we can change this default behavior. We can also inject the properties and their values by using various external options. So these are all the options that we can use. The very first basic option is defining the property and its value inside the application.properties file or application.yml file. Apart from directly defining inside your source code, you can also define the properties and their values by using operating system environment variables or Java system properties or JNDI attributes or ServletContext init params or ServletConfig init parameters or command line arguments. In case if the same property is defined at multiple locations, then the spring boot, it is going to follow an order of precedence. As a spring boot navigates down through these approaches, the priority is going to increase. That means the properties and their values defined inside these application.properties, they are going to have lowest precedence compared to environment variables option. In case if you have defined a property by using both these environment variables and the command line arguments, then the values that we have defined by using the command line arguments is going to be leveraged by the spring boot framework. Because command line arguments has highest precedence compared to environment variables. Please remember this list carefully. As you navigate towards the down, the precedence is going to increase. Developers, usually most of the times they're going to end up defining properties either inside application.properties file or by using environment variables or Java system properties or command line arguments. Other three properties, they are going to be used rarely. That's why we're not going to focus on these three properties inside this course.
