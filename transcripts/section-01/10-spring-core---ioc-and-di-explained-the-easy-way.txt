The first one that we have is Inversion of Control. So Inversion of Control is a software design principle that defines how objects are created and managed in a program. This principle recommends, don't create the objects by yourself, instead, outline a way for their creation and dependency management. Usually, in traditional core Java programs, developers create objects manually by using new keyword. Whereas with IOC, instead of the programmer managing the flow of the application, a framework or a library or a service takes over this responsibility, ensuring objects and their dependency are handled automatically. Inversion of Control is a software design principle that is not only specific to Spring. If you go to any other languages and their frameworks, you will see the same IOC is being used there. Spring also adopted this design principle to make developer life easy. When we are using Spring framework to develop a web application, by using this IOC, the Spring framework, it is going to manage object creation and their dependencies automatically. So with this, what is the advantage that we are going to get? It reduces the manual coding and make applications more flexible. Inversion of Control is about the objects creation. So Dependency Injection is a design pattern used to implement Inversion of Control.
