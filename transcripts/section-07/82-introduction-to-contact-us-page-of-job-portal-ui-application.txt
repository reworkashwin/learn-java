We already discussed about all these details in the previous sections, but let's try to quickly revise what are the steps that we need to complete in order to build a new REST API. The first one, definitely we have to define a controller class on the backend. And inside this controller class, we need to create REST API related Java methods. So on top of these Java methods, we can mention the required annotations like @PostMapping, @GetMapping along with the API path. Next, we need to take care of defining the service layer followed by repository or data layer. So as part of these repository or data layer, we need to create required entity classes and repository interfaces. We should also take care of creating the required database tables for our REST API. At the end of the day, whatever data that UI application is sending to our backend REST API, those information needs to be saved into the database tables or in case if our REST API want to send the data to UI application, then the REST API logic, it is going to read the data from the database tables. In the previous sections, we have created a database table with the name companies. And inside that we have inserted the data followed by we created the entity classes and the repository interfaces. That is one style of building the logic inside the repository or data layer. There is also another style where we can first define the entity class. From these entity class, we can ask the springboard framework to create the required database tables. Both approaches are possible. The traditional approach is first, the database table will be created. From this table, we are going to create all the required classes and interfaces like entity and repository. The reverse approach is also possible where we can first create the entity class. By using these entity class, we can request the framework or the springboard to create the corresponding database table.
