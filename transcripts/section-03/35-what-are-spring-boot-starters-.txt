The Spring Web dependency that we have added in the previous lecture to build a Spring Boot web application, this is not a normal Maven dependency. This Spring Web is a Spring Boot starter. Whatever dependencies that you see inside these websites start.spring.io, we call all these dependencies as Spring Boot starters. Inside this lecture, let's try to understand what are these Spring Boot starters? What are the advantages that they are providing to the developers compared to the normal Maven dependencies? When we add the Spring Web dependency inside our pom.xml, these two dependencies are going to be added. The first one is with the name Spring Boot starter web MVC. The other one is also related to the same web MVC, but this is more of a unit testing dependency. Adding these test dependencies is completely optional. In case if you are not writing any unit testing, you can completely ignore and delete the second dependency. And with that, we are going to left with only one dependency inside our pom.xml with the name spring-boot-starter-webmvc. Usually when you add a Maven dependency, only a given single library will be downloaded into your project classpath. But when we add a Spring Boot starter project, it is going to download all the required ecosystem that is required to build a given feature. For example in our scenario, we want to build a Spring Boot based web application, and that's why we have added these Spring Boot starter project. Behind the scenes this starter project, it knows what are the required libraries to build a production grade web application. And accordingly, all the required libraries, configurations, auto settings, everything will be downloaded and set up inside the Spring Boot project. I can also show you a confirmation to prove you that this is not a normal dependency instead, it is a Spring Boot starter project.
