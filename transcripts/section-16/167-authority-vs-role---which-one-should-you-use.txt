Inside spring security framework, you will hear two different jargons with the name Authority and Role. By using any of them, we should be able to implement authorization, but there is a minor difference between these authority and role. Let's try to understand what is the difference. Authority is like an individual privilege or an action that an end user can perform. By using this authority, we're going to restrict the access in a fine grained manner. For example, we can define authorities like VIEWJOB,ADDJOB,ADDCOMPANY for each and every possible action inside our backend application, we have to define an authority, whereas role is like a group of privileges or actions. By using roles, we can restrict access in a coarse grained manner. Few developers, they are not comfortable defining all the actions that are possible inside their backend applications. Sometimes the number of actions that an end user can perform inside an application can be even in thousands as well, maintaining all those thousand action details and along with their authority details, it is going to be a very cumbersome process. Whereas with the role, the developers and the business analysts, they're going to maintain some data in some documentation like in Excel, where a user with a ROLE_ADMIN can perform so-and-so actions like ADDJOB, DeleteJob View job, whereas for a normal user, they will maintain inside the documentation that the normal user can only view the existing job. So this is how they're going to maintain inside the documentation, but coming to the spring security authorization implementation, they're going to enforce authorization by using simple role like ROLE_ADMIN,ROLE-EMPLOYER  and so on. . By using these roles concept, we can group a list of privileges or actions.
