Whenever we are using this AfterReturningAdvice, we are telling to the Spring framework that we want this aspect to be executed only if the intercepted method is executed successfully without any issues. So whatever response that I have provided here, the same, I have to use it here. Since the framework will not know what is the return type of our method at compilation time, we should make sure we are mentioning the object here instead of object. Maybe in future, I may have multiple login methods for all of them here I can simply mention API login asterisk and with that, I should be able to accept the response as an input here by using this object. But in case if you are expecting the return object from the target method to be used inside your aspect logic, then it is mandatory to use the AfterReturningAdvice.
