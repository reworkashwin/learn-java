What is Spring AOP? AOP is a programming paradigm that aims to increase modularity by allowing the separation of cross-cutting concerns. This AOP concept is not specific to Java ecosystem or Spring ecosystem. The same AOP programming paradigm you will also see in other languages as well. So what are cross-cutting concerns? These are the features required in different parts of an application, but they are not the primary business logic. For example, Logging, any Security validations, any Auditing that we want to perform, any Transaction Management, Caching and so on. None of them are actual business logic. Those are supporting components that are required to run an enterprise application. This kind of non-business logic, we call them as Cross-Cutting Concerns. So this AOP programming paradigm, what it recommends is, it recommends moving all these cross-cutting concerns into a separate location so that these non-business logic can be maintained in a single place. That's what the definition also conveys. It aims to increase the modularity. Modularity means you separate your logics based upon their use case. You're not going to club everything inside a Java method. Without AOP, we saw what is going to happen. The entire code related to logging, any security or transaction management, it is going to get repeatedly mixed with our core business methods. And with that, the primary business methods, they will become cluttered and harder to read or maintain. Apart from maintaining issues, in future, whenever we want to upgrade any of these cross-cutting concerns logic, we will end up revisiting all the Java methods inside an application if someone is not using AOP. Whereas with AOP, we can define all the cross-cutting concerns in a single reusable component or module called Aspect. That's why this name has been given inside the AOP, A means Aspect, Aspect Oriented Programming. Once all the cross-cutting concerns logic defined inside an Aspect, the Spring Framework or the Spring Boot Framework, it is going to view this logic into actual business methods at the specified points based upon our instructions. So this process is called as Weaving. We're going to discuss more details about this weaving in few minutes.
