In the previous lectures we have been talking about type erasure, and in this lecture I would like to tell a few words about quite a common interview question such that why primitive types cannot be used in generics. So why it is not possible to use a list of primitive integers or a list of primitive doubles. By the way, it is a rule of thumb that when dealing with generics related interview questions, most of the answers are related to type erasure. So why it is not possible to use primitive integers? The answer is that because of type erasure. Why? Because we have been talking about that. If we create a generic implementation, a generic class with a generic field variable, then if we do not use a bound but we use something like this, then generics are removed at compile time and it is replaced by object. And this is the answer that essentially objects are the parents of every single reference type. And this is exactly why we can use it with integers, because object is the parent of integer. But int like this is a primitive type. So object is not the parent of the primitive integer. This is why Java cannot use primitive integers when dealing with generics. So we can come to the conclusion that generic works if the generic type is a type that is compatible with the object. So a reference type. Because primitive integer is a primitive, it is not an object. They do not inherit from the object class. So just to summarize again, primitives cannot be used with generics because generics require reference types and primitives are not objects. So this is exactly why we get a compile time error like this. But if we use an integer, it works fine.
