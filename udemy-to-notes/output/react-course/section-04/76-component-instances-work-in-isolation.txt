Now, before we're going to continue working on this player component and this editing functionality, there's one important thing I wanna point out here. As you can see in this application here, if I reload if I click Edit here, we show the input field here but the other player component here does not change. Here we're still showing the player name instead. And that's a pretty important thing to note because in the end of course we are using the same component here, the player component. But whenever you are using or reusing a component, React will basically create a new isolated instance. So even though both these players here use the same player component, they work totally isolated from each other. If the state in this first player component instance here changes, the second player component instance does not care about that at all. It doesn't even know about that. So we have a shared component with the same logic inside of it, but then once you use that component, totally different isolated instances are created which only use the same logic but they then use it on their own. And that's a really important feature in React, and a feature which I absolutely wanna emphasize here because this is one of the most powerful React features. Having this isolation allows you to build super complex reusable components which don't interfere with each other. Because of course here it would not be great if we would click Edit here and the second player would get edited for some reason. That would be horrible, and this whole component concept would be useless if they would behave like this, but thankfully in React, they don't.
