Now there is more, and we will move on to a more advanced example in just a second, but before we do that, there's one other important thing you should know about form actions. You don't have to create these action functions inside of a component function. You can, as you see here, but you don't have to. If in this action function you are not using any component specific data, so if you're not using props or some state in here, you can absolutely take this function and move it out of this component function. So that's what I did here now. I grabbed this signupAction function and I moved it right below my imports outside of the component function so that the component function is leaner. And of course, you could also store this action function in a totally different file if you wanted to do that. It doesn't have to be in the same file as the component. And especially moving it out of the component function can make sense from a performance perspective because if the action function is inside of the component, the action function will of course be recreated every time the component function executes. Now, this is not necessarily a big problem, but it could become a problem depending on the complexity of your application and how often this component function is being re-executed. And therefore moving the action function out of it if you don't need it to be inside of the component function can make sense. Of course, if you were using props or state in that action function, you would have to move it back into the component function so that you have access to those props or state. But here, that's not the case. So moving it out of there makes sense. And with that, we can now move on to the more advanced features.
